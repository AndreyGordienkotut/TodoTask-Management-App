Цей проєкт - мікросервісна система керування завданнями, побудована на архітектурі з декількох незалежних сервісів.
Система дозволяє користувачам створювати, редагувати, видаляти та керувати своїми завданнями через REST API, та отримувати повідомлення(детальніше знизу).
Проєкт являє собою повний бекенд-сервіс, готовий до інтеграції з будь-яким фронтенд-додатком.

Мікросервісна архітектура 

Система розподілена на кілька незалежних сервісів:
- auth-service — реєстрація та авторизація користувачів
- task-service — керування завданнями
- notification-service — відправка повідомлень
- eureka-service — для взаємодії між мікросервісами

Основний функціонал

UserService
- Реєстрація: Створення нового облікового запису користувача
- Авторизація: Вхід в систему за допомогою облікових даних
- Управління Токенами та Сесією:
  - Генерація та валідація Access Token (JWT) для доступу до захищених ресурсів
  - Генерація та використання Refresh Token для отримання нових Access Token без повторної авторизації
  - Вихід із системи (Logout) із анулюванням токенів
- Підтвердження Акаунту: Верифікація користувача через Email
- Інтеграція з Telegram, за для підключення облікового запису користувача до Telegram
  
TaskService
- CRUD(Створення, читання, оновлення та видалення) завдань
- Архівування завдань що вже виконані
- Пошук завдань за назвою або фільтрацією
- Управління статусами: Зміна статусу завдання
- Управління пріоритетами: Встановлення та зміна приоритету завдання
- Планування та перевірка завдань:
  - Планування завдань за разкладом
  - Перевірка завдань, термін виконаня яких наближається або минув
- Генерація подій: Відправка подій про зміни завдання до Kafka для подальшого сповіщення
  
NotificationService
- Споживання Подій: Отримання подій від UserService та TaskService через Kafka
- Відправка повідомлень:
  - Надсилання Email-повідомлень, для підтвердження аккаунту або нагадувань зробити завдання
  - Надсилання Telegram-повідомлень, для нагадування зробити завдання
- Інтеграція з Telegram: Зв'язування облікового запису користувача з Telegram.

Технології що використовувались
- Java 17
- Spring Boot 3 (Web, Data JPA, Security, Validation)
- Spring Cloud (Eureka, OpenFeign, LoadBalancer)
- MySQL
- Gradle
- Docker/ Docker Compose
- Apache Kafka
- JJWT (JSON Web Token)
- GitLab CI/CD
- OpenAPI (Swagger UI)
- JUnit 5, Testcontainers
  
Для запуску репозиторія потрібно
- Встановити Docker та Docker Compose, якщо не встановленний
- Склонувати репозиторій
- Налаштувати зміні середовища(.env файл) для цього:
  - Скопіювати файл .env.example
  - Створити свій файл .env та заповнити його необхідними значеннями за прикладом
- Запустити усі сервіси через команду docker-compose up --build

Додаткова інформація
- Реалізовано за трирівневою архітектурою: Controller → Service → Repository
- Для кожного сервісу було проведена повне модульне та интегроване тестування
- Усі сервіси працюють незалежно та можуть масштабуватись
- Код організовано з урахуванням SOLID принципів
- Використано DTO, Mapper та ResponseEntity для API
